## Data Flow

### Transient Simulation (Web)

```
User clicks "Run"
  вҶ’ app.js calls POST /api/init with nodes[] + links[] JSON
  вҶ’ server.py deserialises, calls ThermalSystem.add_node() / add_link()
  вҶ’ Per time step: POST /api/step with dt, solver type, current params
  вҶ’ server.py calls step_rk4(dt) or step_explicit_euler(dt)
  вҶ’ C++ solver integrates ODEs, returns new temperatures
  вҶ’ app.js updates canvas colour map and Chart.js time plots
```

### Desktop App

```
User edits node/link in ImGui Attribute Sheet
  вҶ’ main.cpp calls solver.update_node() / solver.update_link() directly
  вҶ’ Per frame: solver.step_rk4(dt) called in render loop
  вҶ’ Canvas redrawn with temperature gradients and flow chevrons
  вҶ’ ImPlot panels update in real time
```

## Component Responsibilities

| Component | Responsibility |
|-----------|---------------|
| `solver.cpp` | All physics: ODE integration, fluid properties, fan matching |
| `bindings.cpp` | C-ABI DLL exports for ctypes |
| `thermal_solver.py` | Python-side ctypes wrapper, unit helpers |
| `server.py` | HTTP routing, JSON deserialisation, session state |
| `app.js` | Canvas rendering, simulation loop, UI event handling |
| `main.cpp` | ImGui layout, ImPlot rendering, file I/O, desktop state |
